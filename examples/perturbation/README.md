Milestone-gated scripts building the flagship: predicting perturbed
single-cell expression states from baseline expression + perturbation
identity, trained and benchmarked on the Norman et al. 2019 Perturb-seq
dataset.

## Milestones

Each milestone produces a concrete artifact in [`results/`](results/) and
a findings write-up in [`docs/`](docs/). The progression mirrors the
three-stage architecture (deterministic → self-supervised → uncertainty).

| # | Script | What it produces | Target week | Status |
|---|--------|------------------|-------------|--------|
| **P1** | `P1_download_and_qc.py` | Norman 2019 loaded, QC'd, train/val/holdout splits | Week 1 | 🎯 in progress |
| **P2** | `P2_cvae_nb_baseline.py` | CVAE_NB with perturbation conditioning; Pearson ≈ 0.8 target (matches scGen) | Week 1-2 | 📝 planned |
| **P3** | `P3_jepa_predictor.py` | JEPA with VICReg; latent-space prediction of perturbed state | Week 3-4 | 📝 planned |
| **P4** | `P4_latent_diffusion.py` | Latent diffusion wrapper on JEPA latent; uncertainty quantification | Week 5 | 🔮 planned |
| **P5** | `P5_benchmark_suite.py` | Full benchmark: our stack vs scGen / CPA / scPPDM | Week 6 | 🔮 planned |

Status legend: 🎯 active · 📝 prototype planned · 🔮 pending predecessor.

## Running a Milestone

### Local (smoke testing, small subsets)

```bash
mamba activate genailab
python examples/perturbation/P1_download_and_qc.py --help
python examples/perturbation/P1_download_and_qc.py          # full run
```

Each script supports `--help` and writes to `results/` + optionally
`--wandb` for training runs.

### Remote (full-scale training)

```bash
# From project root
python ops/provision_cluster.py --stage-data \
    --data-path scrna/perturb_seq/norman_2019    # first time only

python ops/provision_cluster.py                  # subsequent
ssh genai-workspace
cd /workspace/genai-lab
python examples/perturbation/P2_cvae_nb_baseline.py --wandb --epochs 100
```

See [`examples/docs/running_experiments.md`](../docs/running_experiments.md)
for the full nohup + PID + rsync pattern, and
[`examples/docs/experiment_tracking.md`](../docs/experiment_tracking.md)
for the W&B workflow.

## Design Conventions

- **Each milestone is self-contained** — `Px_*.py` should run end-to-end
  given a fresh environment. Shared logic lives in
  `src/genailab/applications/perturbation/`.
- **Raw counts are non-negotiable for NB/ZINB decoders** — see
  [`CLAUDE.md`](../../CLAUDE.md) Domain Conventions. Never normalize before
  passing to the decoder.
- **Library size is a covariate, not a preprocessing step** — computed on
  the full filtered gene set before HVG subsetting, passed explicitly to
  the decoder.
- **W&B runs tagged `perturbation`** — use `--wandb-tags perturbation,Pn`
  so sweep comparisons filter cleanly.
- **Results artifacts stay under `results/`** — never commit trained
  weights; they live under [`runs/perturbation/`](../../runs/) with W&B
  artifact metadata.

## Benchmark Targets (for P2 onwards)

| Method | Published Pearson on held-out perturbations | Our target |
|--------|---------------------------------------------|------------|
| scGen (Lotfollahi 2019) | ~0.80 | match (P2 baseline) |
| CPA (Lotfollahi 2023) | ~0.85 | match (P3 JEPA) |
| scPPDM (2025) | ~0.87 | match/exceed (P4 diffusion) |
| GEARS (Roohani 2024) | ~0.82 (different eval) | comparison only |

See the methodology write-up in
[`docs/applications/perturbation_prediction.md`](../../docs/applications/perturbation_prediction.md)
for evaluation details, compositional-generalization protocol, and
biology-validation checks (DEG recovery, pathway consistency).
