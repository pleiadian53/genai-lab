# Perturbation Imaging — Production Application

Milestone-gated scripts predicting **phenotypic perturbation response**
from cell-painting microscopy data. The image-space sibling to the
scRNA-seq perturbation flagship: same perturbation embedding, different
output modality.

```
perturbation_id  →  predicted cell image  (latent DiT, this directory)
perturbation_id  →  predicted gene expression  (CVAE_NB / JEPA / latent diffusion, examples/perturbation/)
```

See also:
- [`dev/planning/perturbation_imaging_path.md`](../../../dev/planning/perturbation_imaging_path.md) — durable plan, open decisions, principles
- [`examples/perturbation/README.md`](../../perturbation/README.md) — sibling scRNA-seq application

## Architecture

```
real image → VAE encoder → latent z → VelocityDiT2D → v_θ(z, t, c) → ODE in z-space → VAE decoder → generated image
                                            ▲
                                   perturbation embedding c
```

The DiT and flow-matching machinery are shared with the POC under
`notebooks/vision_models/`. New work here is the cell-painting data
pipeline, the **domain-specific multi-channel VAE**, the perturbation
conditioning module, and the production-grade evaluation suite.

---

## Production-from-day-one principles

These shape every milestone in this directory:

1. **Real public data only** — no synthetic stand-ins
2. **Domain-specific preprocessing** is its own milestone, not a footnote
3. **Domain-specific VAE** trained on the target dataset (cell painting
   is 5–6 channels — pretrained natural-image VAEs cannot be used here)
4. **Headline metric**: perturbation-classifier accuracy on synthetic
   samples, not FID
5. **Reproducibility scaffolding from V1** — seeds, configs, checkpoint
   metadata, W&B tagging
6. **`ops/`-staged from V2** — training does not run on a laptop

Full rationale in [`dev/planning/perturbation_imaging_path.md`](../../../dev/planning/perturbation_imaging_path.md).

## Milestones

Each milestone produces one artifact in [`results/`](results/) and one
findings doc in [`docs/`](docs/).

| # | Script | What it produces | Compute | Status |
|---|--------|------------------|---------|--------|
| **V1** | `V1_download_and_qc.py` | Cell-painting subset (RxRx1 default), illumination-corrected, plate-normalized, train/val/test splits | CPU | 🔮 planning |
| **V2** | `V2_train_domain_vae.py` | Multi-channel KL-VAE; per-channel reconstruction PSNR/SSIM; biology check (perturbation cluster preservation in latent) | A40 | 🔮 planning |
| **V3** | `V3_dit_unconditional.py` | Unconditional `VelocityDiT2D` on V2 latents; sample grid that visually resembles real cells | A40 | 🔮 planning |
| **V4** | `V4_dit_perturbation.py` | Conditional `VelocityDiT2D` with perturbation embedding; samples that visibly vary across perturbations | A40 | 🔮 planning |
| **V5** | `V5_benchmark.py` | Headline: perturbation-classifier accuracy on synthetic vs real. FID secondary. CVAE-image baseline. | A40 | 🔮 planning |

Status legend: 🔮 planned · 📝 in progress · 🎯 active · 🔬 validated · ✅ mature.

## Running a milestone (template, no scripts yet)

### Local (data download + QC only)

```bash
mamba activate genailab
python examples/vision_models/perturbation_imaging/V1_download_and_qc.py --help
```

### Remote (V2 onwards — GPU required)

```bash
# First time only
python ops/provision_cluster.py --stage-data \
    --data-path imaging/rxrx1

python ops/provision_cluster.py
ssh genai-workspace
cd /workspace/genai-lab
python examples/vision_models/perturbation_imaging/V2_train_domain_vae.py --wandb --epochs 100
```

## Library home

Reusable components live under `src/genailab/vision/` (not nested under
`flow_matching/`, since future vision work — MAE, vision FMs — won't fit
cleanly there):

```
src/genailab/vision/
├── data/
│   └── cell_painting.py            # loader + QC primitives
├── vae/
│   └── multichannel_kl_vae.py      # V2 architecture
├── networks/
│   └── velocity_dit_2d.py          # shared with notebooks/vision_models/
└── conditioning/
    └── perturbation_embed.py       # V4 module
```

Created incrementally as each V-milestone needs it.
