2. Architecture Overview

The Vision Transformer (ViT) adapts the encoder-only Transformer from NLP to image classification. The key insight: an image can be treated as a sequence of patch "tokens", just as a sentence is a sequence of word tokens.

3. Patch Embedding

3.1 Conceptual view: image → sequence of tokens

Given an input image x∈RH×W×Cin\mathbf{x} \in \mathbb{R}^{H \times W \times C_{\text{in}}}x∈RH×W×Cin​ and patch size PPP:

Step 1 — Divide into non-overlapping patches

Partition the image into a grid of N=HP×WPN = \frac{H}{P} \times \frac{W}{P}N=PH​×PW​ patches, each of shape (P,P,Cin)(P, P, C_{\text{in}})(P,P,Cin​). For our mini ViT: N=324×324=8×8=64N = \frac{32}{4} \times \frac{32}{4} = 8 \times 8 = 64N=432​×432​=8×8=64 patches.

Step 2 — Flatten each patch to a vector
xip∈RP×P×Cin  →flatten  xiflat∈RP2⋅Cin\mathbf{x}_i^p \in \mathbb{R}^{P \times P \times C_{\text{in}}} \;\xrightarrow{\text{flatten}}\; \mathbf{x}_i^{\text{flat}} \in \mathbb{R}^{P^2 \cdot C_{\text{in}}}xip​∈RP×P×Cin​flatten​xiflat​∈RP2⋅Cin​

Each 4×4×34 \times 4 \times 34×4×3 patch becomes a 48-dimensional vector.

Step 3 — Linear projection to embedding space
zi=E⋅xiflat+b,E∈R(P2⋅Cin)×D\mathbf{z}_i = \mathbf{E} \cdot \mathbf{x}_i^{\text{flat}} + \mathbf{b}, \qquad \mathbf{E} \in \mathbb{R}^{(P^2 \cdot C_{\text{in}}) \times D}zi​=E⋅xiflat​+b,E∈R(P2⋅Cin​)×D

Each 48-dim flattened patch is projected to a D=128D = 128D=128 dimensional embedding. The projection matrix E\mathbf{E}E and bias b\mathbf{b}b are learnable — the model learns what features to extract from each patch.

3.2 The Conv2d trick

Rather than manually splitting, flattening, and projecting, we use a single Conv2d with kernel_size = stride = P: 

self.projection = nn.Conv2d(
    in_channels=3,       # C_in: RGB
    out_channels=128,    # D: embedding dim
    kernel_size=4,       # P: covers exactly one patch
    stride=4,            # P: no overlap between patches
)

This is mathematically equivalent to flatten + linear:

  • The 4×44 \times 44×4 kernel covers exactly one patch (no overlap because stride=kernel_size\text{stride} = \text{kernel\_size}stride=kernel_size).
  • Each of the 128 output filters computes one dimension of the patch embedding.
  • The conv output has shape (B,D,H/P,W/P)=(B,128,8,8)(B, D, H/P, W/P) = (B, 128, 8, 8)(B,D,H/P,W/P)=(B,128,8,8).

We then reshape to get a sequence:

(B,D,HP,WP)  →flatten(2)  (B,D,N)  →transpose(1,2)  (B,N,D) (B, D, \tfrac{H}{P}, \tfrac{W}{P}) \;\xrightarrow{\texttt{flatten(2)}}\; (B, D, N) \;\xrightarrow{\texttt{transpose(1,2)}}\; (B, N, D) (B,D,PH​,PW​)flatten(2)​(B,D,N)transpose(1,2)​(B,N,D)
Why use Conv2d? It avoids explicit reshape operations and leverages highly optimised GEMM kernels. The equivalence holds because a convolution with stride=kernel_size\text{stride} = \text{kernel\_size}stride=kernel_size computes independent dot products at non-overlapping positions — exactly what a linear layer on flattened patches would do.

4. CLS Token

A learnable vector zcls∈RD\mathbf{z}_{\text{cls}} \in \mathbb{R}^{D}zcls​∈RD is prepended to the sequence of patch tokens:

Z0=[ zcls  ∣  z1, z2, …, zN ]∈R(N+1)×D \mathbf{Z}_0 = \bigl[\,\mathbf{z}_{\text{cls}} \;\big|\; \mathbf{z}_1,\, \mathbf{z}_2,\, \dots,\, \mathbf{z}_N\,\bigr] \quad \in \mathbb{R}^{(N+1) \times D} Z0​=[zcls​​z1​,z2​,…,zN​]∈R(N+1)×D

After passing through all LLL transformer blocks, the CLS token's output (position 0) serves as the image representation for classification. Through self-attention, information from all 64 patches is aggregated into this single token.

Implementation detail: broadcasting + cat

# cls_token is nn.Parameter of shape (1, 1, D)
cls_tokens = self.cls_token.expand(B, -1, -1)  # (1,1,D) → (B,1,D) via expand (zero-copy)
tokens = torch.cat([cls_tokens, tokens], dim=1) # (B,1,D) + (B,N,D) → (B, N+1, D)

The .expand() broadcasts the single learnable CLS vector across the batch dimension without copying memory (stride-0 view). The torch.cat along dim=1 then prepends it to every sample's patch sequence.

Alternative: global average pooling. Instead of a CLS token, some ViT variants average-pool all NNN patch token outputs: zˉ=1N∑i=1Nzi(L)\bar{\mathbf{z}} = \frac{1}{N}\sum_{i=1}^{N} \mathbf{z}_i^{(L)}zˉ=N1​∑i=1N​zi(L)​. Both approaches work; the CLS token is the original ViT design, inherited from BERT.

5. Positional Embedding

Self-attention is permutation-equivariant: if you shuffle the input tokens, the output tokens are shuffled in exactly the same way. Without positional information, the model cannot distinguish patch (0,0)(0, 0)(0,0) (top-left) from patch (7,7)(7, 7)(7,7) (bottom-right).

ViT adds learnable positional embeddings:

Z0′=Z0+Epos,Epos∈RN′×D \mathbf{Z}_0' = \mathbf{Z}_0 + \mathbf{E}_{\text{pos}}, \qquad \mathbf{E}_{\text{pos}} \in \mathbb{R}^{N' \times D} Z0′​=Z0​+Epos​,Epos​∈RN′×D

Each of the N′=N+1=65N' = N + 1 = 65N′=N+1=65 positions (1 CLS + 64 patches) gets its own learnable DDD-dimensional vector. The addition is element-wise — each token embedding is shifted by its position's vector.

Implementation detail: broadcasting via +

# pos_embed is nn.Parameter of shape (1, N', D) = (1, 65, 128)
tokens = tokens + self.pos_embed
# Broadcasting: (B, 65, 128) + (1, 65, 128) → (B, 65, 128)
# The batch dim broadcasts automatically (element-wise op)

Why 1D and not 2D positional embeddings?

The patches have a 2D spatial arrangement (8×88 \times 88×8), yet we use a flat 1D index (0,1,…,64)(0, 1, \dots, 64)(0,1,…,64). The original ViT paper found that learnable 1D positional embeddings perform just as well as 2D-aware variants. The model discovers the 2D grid structure from data — you can verify this by computing cosine similarity between positional embeddings and observing that nearby patches have similar embeddings (see notebook Section 11).

Sinusoidal vs learnable. Unlike the original Transformer (Vaswani et al., 2017) which used fixed sinusoidal encodings, ViT uses learnable positional embeddings initialised from N(0,0.022)\mathcal{N}(0, 0.02^2)N(0,0.022). For the short sequences in vision (N′=65N' = 65N′=65), learnable embeddings have enough parameters to be learned effectively without the inductive bias of sinusoids.

6. Multi-Head Self-Attention (MHSA)

Self-attention is the mechanism by which each patch token gathers information from every other patch token. This global receptive field from the first layer is a key structural difference from CNNs, where receptive fields grow gradually with depth.

6.1 Single-head scaled dot-product attention

Given an input sequence Z∈RN′×D\mathbf{Z} \in \mathbb{R}^{N' \times D}Z∈RN′×D:

Step 1 — Project to queries, keys, and values
Q=Z WQ,K=Z WK,V=Z WVQ = \mathbf{Z}\, W_Q, \qquad K = \mathbf{Z}\, W_K, \qquad V = \mathbf{Z}\, W_VQ=ZWQ​,K=ZWK​,V=ZWV​

where WQ,WK,WV∈RD×dkW_Q, W_K, W_V \in \mathbb{R}^{D \times d_k}WQ​,WK​,WV​∈RD×dk​. Queries encode "what I'm looking for", keys encode "what I contain", and values encode "what I provide if attended to".

Step 2 — Compute attention scores
A=softmax⁡ ⁣(QK⊤dk)∈RN′×N′A = \operatorname{softmax}\!\left(\frac{Q K^\top}{\sqrt{d_k}}\right) \quad \in \mathbb{R}^{N' \times N'}A=softmax(dk​​QK⊤​)∈RN′×N′

Element Ai,jA_{i,j}Ai,j​ is how much token iii attends to token jjj. Each row sums to 1 (softmax normalisation). The dk\sqrt{d_k}dk​​ scaling prevents dot products from growing too large as dkd_kdk​ increases — without it, softmax saturates (outputs near 0 or 1) and gradients vanish.

Step 3 — Aggregate values
head=A⋅V∈RN′×dk\text{head} = A \cdot V \quad \in \mathbb{R}^{N' \times d_k}head=A⋅V∈RN′×dk​

Each output token is a weighted sum of all value vectors, weighted by attention. Token iii's output is: ∑j=1N′Ai,j⋅Vj\sum_{j=1}^{N'} A_{i,j} \cdot V_j∑j=1N′​Ai,j​⋅Vj​.

6.2 Why multiple heads?

A single attention head can only express one "type" of relationship (e.g., spatial proximity). Multiple heads allow the model to simultaneously attend to different aspects (colour similarity, edge alignment, semantic relatedness, etc.).

With hhh heads, each operating on dk=D/hd_k = D/hdk​=D/h dimensions:

MHSA⁡(Z)=Concat⁡(head1, head2, …, headh)⋅WO \operatorname{MHSA}(\mathbf{Z}) = \operatorname{Concat}(\text{head}_1,\, \text{head}_2,\, \dots,\, \text{head}_h) \cdot W_O MHSA(Z)=Concat(head1​,head2​,…,headh​)⋅WO​

where WO∈R(h⋅dk)×D=RD×DW_O \in \mathbb{R}^{(h \cdot d_k) \times D} = \mathbb{R}^{D \times D}WO​∈R(h⋅dk​)×D=RD×D projects the concatenated heads back to DDD dimensions.

6.3 The efficient multi-head reshape trick

Rather than using hhh separate Linear(D, d_k) layers, we compute Q, K, V with a single combined projection and reshape to split heads: 

# Combined QKV projection: one Linear instead of three
self.qkv = nn.Linear(D, 3 * D)     # (B, N', D) → (B, N', 3D)

# In forward():
qkv = self.qkv(x)                                      # (B, N', 384)
qkv = qkv.reshape(B, N, 3, h, d_k)                     # (B, N', 3, 4, 32)
qkv = qkv.permute(2, 0, 3, 1, 4)                       # (3, B, 4, N', 32)
q, k, v = qkv.unbind(0)                                 # each: (B, 4, N', 32)

The reshape (B,N′,3D)→(B,N′,3,h,dk)(B, N', 3D) \to (B, N', 3, h, d_k)(B,N′,3D)→(B,N′,3,h,dk​) splits the 384-dim output into 3 groups (Q/K/V) × 4 heads × 32 dims per head. The permute reorders axes so that batch and head dims come first, enabling batched @ (matmul) across all heads simultaneously.

7. Feed-Forward MLP

Each transformer block contains a position-wise MLP — a two-layer network applied independently and identically to every token:

MLP⁡(z)=W2⋅GELU⁡(W1⋅z+b1)+b2 \operatorname{MLP}(\mathbf{z}) = W_2 \cdot \operatorname{GELU}(W_1 \cdot \mathbf{z} + \mathbf{b}_1) + \mathbf{b}_2 MLP(z)=W2​⋅GELU(W1​⋅z+b1​)+b2​

where W1∈RD×DmlpW_1 \in \mathbb{R}^{D \times D_{\text{mlp}}}W1​∈RD×Dmlp​ expands the representation (128→256128 \to 256128→256) and W2∈RDmlp×DW_2 \in \mathbb{R}^{D_{\text{mlp}} \times D}W2​∈RDmlp​×D compresses it back (256→128256 \to 128256→128).

Why the expansion?

Attention mixes information across tokens but only via linear operations (projection + weighted sum). The MLP provides non-linear, per-token computation. The expansion to Dmlp>DD_{\text{mlp}} > DDmlp​>D gives the network a wider space to compute non-linear features before projecting back. The standard expansion ratio is 4×4\times4×; we use 2×2\times2× to keep the model small.

Why GELU and not ReLU?

GELU (Gaussian Error Linear Unit) is defined as:

GELU⁡(x)=x⋅Φ(x) \operatorname{GELU}(x) = x \cdot \Phi(x) GELU(x)=x⋅Φ(x)

where Φ(x)\Phi(x)Φ(x) is the CDF of the standard normal distribution. Unlike ReLU which hard-zeros negative inputs, GELU provides a smooth transition. This has been empirically shown to improve training stability for transformers (used in BERT, GPT, ViT).

8. Transformer Encoder Block

Each block applies two sub-layers with pre-norm residual connections:

z′=MHSA⁡ ⁣(LN⁡(z))+z \mathbf{z}' = \operatorname{MHSA}\!\bigl(\operatorname{LN}(\mathbf{z})\bigr) + \mathbf{z} z′=MHSA(LN(z))+z z′′=MLP⁡ ⁣(LN⁡(z′))+z′ \mathbf{z}'' = \operatorname{MLP}\!\bigl(\operatorname{LN}(\mathbf{z}')\bigr) + \mathbf{z}' z′′=MLP(LN(z′))+z′

Pre-norm vs post-norm

The original Transformer (Vaswani et al., 2017) used post-norm: normalise after the sub-layer and residual addition (LN⁡(z+sublayer(z))\operatorname{LN}(\mathbf{z} + \text{sublayer}(\mathbf{z}))LN(z+sublayer(z))). ViT uses pre-norm: normalise before the sub-layer (sublayer(LN⁡(z))+z\text{sublayer}(\operatorname{LN}(\mathbf{z})) + \mathbf{z}sublayer(LN(z))+z).

Pre-norm is preferred because the residual path carries unnormalised gradients directly to earlier layers. In post-norm, every gradient must pass through a LayerNorm, which can dampen the signal. Pre-norm stabilises training, especially for deeper models.

Residual connections

The +z+ \mathbf{z}+z additions create skip paths that allow gradients to flow backward through the network without passing through non-linearities. At initialisation (when sub-layer outputs are near zero), the block approximately computes the identity function z′′≈z\mathbf{z}'' \approx \mathbf{z}z′′≈z, ensuring stable early training.

10. Training Considerations

10.1 Optimiser: AdamW

Transformers are typically trained with AdamW — Adam with decoupled weight decay. Standard Adam applies weight decay through the gradient (equivalent to L2 regularisation), which interacts poorly with adaptive learning rates. AdamW applies weight decay directly to the weights, independent of the gradient scaling:

θt+1=θt−η(m^tv^t+ϵ+λ θt) \theta_{t+1} = \theta_t - \eta \left( \frac{\hat{m}_t}{\sqrt{\hat{v}_t} + \epsilon} + \lambda \, \theta_t \right) θt+1​=θt​−η(v^t​​+ϵm^t​​+λθt​)

where m^t\hat{m}_tm^t​ and v^t\hat{v}_tv^t​ are bias-corrected first and second moment estimates, and λ\lambdaλ is the weight decay coefficient (here λ=0.05\lambda = 0.05λ=0.05).

10.2 Learning rate schedule: warmup + cosine decay

10.3 Gradient clipping

Attention can produce sharp gradients (especially early in training when scores are large). Gradient clipping caps the global gradient norm:

g←g⋅min⁡ ⁣(1,  max_norm∥g∥) \mathbf{g} \leftarrow \mathbf{g} \cdot \min\!\left(1,\; \frac{\text{max\_norm}}{\|\mathbf{g}\|}\right) g←g⋅min(1,∥g∥max_norm​)

This prevents destructive parameter updates without changing the gradient direction.

10.4 ViT on small datasets — expectations

ViTs are data-hungry. Unlike CNNs, which have strong inductive biases (translation equivariance via convolutions, locality via small kernels), ViTs must learn spatial structure from scratch. On CIFAR-10 with 50k images and no pretraining, expect ∼\sim∼70–80% test accuracy — significantly below a ResNet. The original ViT paper showed competitive results only when pretrained on large datasets (ImageNet-21k, JFT-300M). Data augmentation partially compensates but cannot fully close the gap.

11.1 Visualising CLS attention

Different heads attend to different things — one might focus on edges, another on colour regions, another on the object centre. This is exactly why multi-head attention is more expressive than single-head.

13. Extensions and Modern Variants

ExtensionKey ideaReference
DeiT Data-efficient training recipes (distillation token, strong augmentation) that make ViT competitive on ImageNet without massive pretraining. Touvron et al., 2021
Swin Transformer Hierarchical ViT with shifted windows — patches are merged at each stage (reducing NNN), and attention is computed locally within windows. Introduces relative positional bias instead of absolute embeddings. Liu et al., 2021
MAE Masked Autoencoder — self-supervised pretraining where 75% of patches are masked and the model learns to reconstruct them. Analogous to BERT's masked language modelling. He et al., 2022
DiT Diffusion Transformer — replaces the UNet backbone in diffusion models with a ViT-like architecture. Uses adaLN (FiLM-style conditioning) instead of cross-attention for time/class conditioning. Peebles & Xie, 2023
Register tokens Extra learnable tokens (like CLS but not used for classification) that absorb attention artefacts — high-norm tokens that appear in low-information background patches. Darcet et al., 2024
